## Agent failure modes observed in the 2026-04-22 round — counter these in new issue bodies

Every problem below was a sub-20-minute fix in review but cost a CI cycle. Pre-empt them.

1. **TS 5.7 `Uint8Array<ArrayBufferLike>` → DOM `BodyInit` / `BlobPart` mismatch.** When you hand bytes to `new Response(...)` or `new Blob([...])`, TypeScript can't prove the buffer isn't `SharedArrayBuffer`. Either cast (`as unknown as BodyInit`) with a one-line comment, or have the producer return an `ArrayBuffer`. PR #28 hit this on JSZip output.

2. **`vi.fn()` return typing doesn't narrow.** `let mock: ReturnType<typeof vi.fn>` widens to `Mock<Procedure | Constructable>` which won't assign to an optional function slot. Use `let mock: Mock<(args: X) => Y>` (`import type { Mock } from 'vitest'`) or pass the generic to `vi.fn<F>()`. PR #30 hit this in `voice-realtime-client.test.ts`.

3. **Regex overreach in character classes.** `.replace(/[#>*_\`-]/g, '')` strips every `-`, including compound-word hyphens (`golden-hour` → `goldenhour`). Strip bullet/heading markers with an anchored pattern (`/^\s*[-*+>#]+\s*/gm`), then clean inline emphasis separately. PR #31 hit this in README paragraph cleaning.

4. **Extractor pattern mismatch vs real-world data.** `extractFontFamiliesFromCss` only matches `font-family:` declarations; plain-text brand docs ("Display: \"Canela Deck\"") need their own regex. Same for voice samples — don't assume HTML markup on payloads that are just markdown / plain text. PR #31 hit this in `extractFromFiles`.

5. **E2E that assumes tldraw selection state.** PR #29's e2e does `Ctrl+A` after placing an image and expects `getByRole('button', { name: /remove bg/i })` to appear. That chain is flaky / environment-dependent in Playwright. If your slice touches tldraw selection, prefer driving selection through `window.editor.setSelectedShapes([id])` via `page.evaluate` rather than keyboard shortcuts.

6. **"Skeleton + one smoke test" scope drift.** PR #30 declared `'gemini-live'` in a type union but didn't create `lib/voice/gemini-live.ts`. If the issue asks for a skeleton, the file must exist — even if it's a stub that throws `not implemented yet`. Forward-declared types without backing files are scope gaps.

7. **Tool / bot mentions.** No commits or PR bodies this round referenced other tools by name; keep it that way. `Co-Authored-By: Claude Opus 4.7` is the only acceptable attribution line.

8. **Stale handoff doc trap.** The 2026-04-22 handoff listed P0s that had already shipped. Agents that read it verbatim wasted turns re-planning already-done work. **Base every slice off `git log --oneline -20` on `main`**, not off a handoff's static list.
